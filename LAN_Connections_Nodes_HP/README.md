# LAN_Connections_Nodes_HP

Dedicated Cursor workspace on **HP** for **Empire LAN / SMB mesh** between HP and workers (e.g. **AI_X1**).  
**Naming:** suffix **`_HP`** on this node; on **AI_X1** use **`LAN_Connections_Nodes_AI_X1`** (or equivalent `_*_AI_X1`) so project names are not confused across machines.

## 2. Topology (mental model)

```
HP (control)                         Worker (e.g. AI_X1)
C:\Empire                            C:\Empire_AI_X1
     |                                      |
     |  SMB "Z:" (HP maps worker share)     |
     +--------------------------------------+
     |  SMB "Y:" (worker maps HP share)     |
     +--------------------------------------+
```

- **Z:** on HP → `\\<worker IPv4 or AI-X1>\Empire_AI_X1` (or nested layout — see candidates JSON).
- **Y:** on AI_X1 → HP share (e.g. `\\HP-SPECTRE\Empire` or your actual share name) — script: `Set_Empire_LAN_Link_To_HP.ps1` on the worker.

**Bidirectional** means **both** directions work, not only Z:.

## 4. Scripts — primary runbook (HP)

Run from **Admin PowerShell** when fixing links (adjust paths if needed).

### 4.1 Establish / repair HP → worker (Z:)

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\Empire\Scripts\Set_Empire_LAN_Link.ps1"
```

With explicit worker IP (from **ipconfig on worker**):

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\Empire\Scripts\Set_Empire_LAN_Link.ps1" -WorkerIp 192.168.1.38
```

**Behaviors (recent hardening):**

- **Auto-heal:** If `Z:` pointed at **this PC’s own IPv4**, it is **disconnected** automatically (self-map is invalid as “worker”).
- **Auto-resolve after `-WorkerIp`:** If the given IP does not expose any candidate share, the script can **fall back** to DNS / `EMPIRE_LAN_TRY_IPS` / last-known files / optional slow scan — **excluding** the failed IP so it does not loop forever. Opt out: **`-NoFallbackAutoResolve`**.
- **Slow subnet scan:** Off by default (avoids “stuck” feeling). Enable: **`-AllowSlowLanScan`** or `EMPIRE_LAN_ALLOW_SLOW_LAN_SCAN=1`.

### 4.2 One-shot heal (HP)

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\Empire\Scripts\Empire_Node_Mesh_Heal.ps1" -NodeRole HP
```

### 4.3 Directory symlink `C:\Empire_AI_X1` → worker UNC (HP, Admin)

Uses **`mklink /D`** to a real UNC (not junction to `Z:`).

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\Empire\Scripts\Empire_Ensure_HP_EmpireAiX1_Junction.ps1"
```

### 4.4 Push mesh scripts to worker `Scripts` share

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\Empire\Scripts\Push_Empire_Mesh_Scripts_To_AI_X1.ps1"
```

Requires reachable UNC to `...\Scripts` per candidates JSON. Successful run appends **EMPIRE_MIRROR_REGISTER**.

### 4.5 Diagnose

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\Empire\Scripts\Diagnose_Empire_HP_To_AI_X1_Lan.ps1" -WorkerIp 192.168.1.38
```

### 4.6 Worker publishes its LAN IP for HP (run **on AI_X1**)

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\Empire\Scripts\Empire_Publish_Worker_LanIp_ForHp.ps1"
```

(Use the path on the worker — often `C:\Empire_AI_X1\Scripts\...` after push.)

### 4.7 Credentials (optional)

Machine or User environment variables:

- `EMPIRE_LAN_USER`
- `EMPIRE_LAN_PASS`

User scope overrides Machine. Script defaults still exist if unset.

## 5. Worker (AI_X1) checklist

On **AI_X1**, not HP:

1. **Private** network profile; firewall allows **SMB (TCP 445)** on Private.
2. Share **one** of the layouts expected by `EMPIRE_WORKER_SMB_CANDIDATES.json` (e.g. share name **`Empire_AI_X1`** mapping to repo root with **`Scripts`** inside, or nested `Empire\Empire_AI_X1\Scripts`).
3. **Reverse link to HP:** run mesh heal for worker:  
   `Empire_Node_Mesh_Heal.ps1 -NodeRole AI_X1` (uses `Set_Empire_LAN_Link_To_HP.ps1`).
4. **Sibling docs folder:** create **`LAN_Connections_Nodes_AI_X1`** (or similar) with the same structure as this project but **node-local** IPs and screenshots.
